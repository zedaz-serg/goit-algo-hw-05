# Висновки щодо ефективності алгоритмів пошуку підрядка

## 1. Загальні умови експерименту
- Використано три алгоритми: **Боєра–Мура (BM)**, **Кнута–Морріса–Пратта (KMP)** та **Рабіна–Карпа (RK)**.
- Тестування проводилось на двох текстах: *стаття 1* та *стаття 2*.
- Для кожного тексту виконувались два види пошуку:
  - **існуючий підрядок**: `"розгорнутий список"`;
  - **вигаданий підрядок**: `"неіснуючий підрядок"`.
- Час виконання вимірювався за допомогою модуля `timeit`.

---

## 2. Результати для кожного тексту

### Стаття 1
- **Існуючий підрядок**: найшвидшим виявився алгоритм **Кнута–Морріса–Пратта (KMP)**, завдяки ефективній побудові префікс-функції.
- **Вигаданий підрядок**: найшвидшим став **Боєра–Мура (BM)**, оскільки він робить великі стрибки по тексту при відсутності збігів.
- **Рабіна–Карпа (RK)** показав стабільні, але дещо повільніші результати через обчислення хешів.

**Висновок для статті 1:**  
KMP оптимальний для пошуку існуючих підрядків, BM — для вигаданих.

---

### Стаття 2
- **Існуючий підрядок**: найкращий час показав **KMP**, особливо на довгих текстах, де його лінійна складність проявляється найкраще.
- **Вигаданий підрядок**: знову найшвидший **BM**, який ефективно пропускає великі ділянки тексту.
- **RK** залишився найповільнішим через накладні витрати на хешування.

**Висновок для статті 2:**  
KMP найкраще працює для реальних збігів, BM — для вигаданих рядків.

---

## 3. Загальний висновок
- **Кнута–Морріса–Пратта (KMP)** — найшвидший алгоритм для пошуку існуючих підрядків у тексті.  
- **Боєра–Мура (BM)** — найшвидший для випадків, коли підрядок відсутній.  
- **Рабіна–Карпа (RK)** — універсальний, але поступається за швидкістю двом іншим алгоритмам.  

Таким чином:
- Для практичних задач, де потрібно знайти реальні збіги, варто використовувати **KMP**.  
- Для задач, де часто перевіряються неіснуючі рядки, ефективніший **BM**.  
- **RK** доцільний у випадках множинного пошуку різних підрядків одночасно, але не є оптимальним за швидкістю.
